# Account onboarding design

## Goal

友人が手引書から迷わずユーザー登録し、端末間で同じ記録を利用できる状態にする。

## Flow

1. 手引書の主ボタンを「無料でユーザー登録」にする。
2. `?start=account` 付きでアプリを開き、登録・ログイン画面を自動表示する。
3. メールアドレスを送信し、届いたリンクを同じ端末で開く。
4. Supabase Authで初回は登録、以降はログインとして扱う。
5. ログイン後は既存のクラウド同期処理を使い、デッキ登録へ進む。

## UI principles

- 通常利用者にはSupabase URLやPublishable keyを入力させない。
- 登録とログインを分けず、メール入力を1回にする。
- パスワード不要であることと、メール内リンクを開く次の行動を明示する。
- 登録前でもローカル利用は維持するが、手引書では登録を推奨する。

## Safety

- ブラウザにはPublishable keyだけを設定する。
- Secret keyとservice role keyは使用しない。
- データ分離は既存のRLSと`user_id`制約を継続利用する。
